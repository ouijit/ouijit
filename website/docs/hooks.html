Hooks

Script hooks and Claude Code event integration for automating your workflow.

Script hooks

Hooks are shell scripts that Ouijit runs in response to events. They let you automate repetitive tasks like running linters, restarting dev servers, or sending notifications.

Configuring hooks

Open the hook configuration dialog from your project settings. Each hook has:

  • Event — when the hook runs
  • Script — the shell command to execute
  • Timeout — maximum execution time (default: 30 seconds)

Available events

EventWhen it fires
task.startA task moves to In Progress and its worktree is created
task.completeA task moves to Done
terminal.openA new terminal session is opened
terminal.closeA terminal session is closed

Environment variables

Hooks receive context about the event via environment variables:

OUIJIT_TASK_NUMBER=42
OUIJIT_TASK_TITLE="Add user auth"
OUIJIT_WORKTREE_PATH=/path/to/worktrees/my-project/T-42
OUIJIT_BRANCH=T-42
OUIJIT_PROJECT_PATH=/path/to/my-project

Claude Code integration

Ouijit runs an HTTP server that receives status events from Claude Code's hook system. This powers the real-time agent status indicators in the UI.

How it works

  1. When a terminal opens, Ouijit injects Claude Code hook configuration pointing to its local HTTP server.
  2. As Claude Code works, it sends status events (thinking, writing code, running commands) to Ouijit.
  3. Ouijit displays these events in the terminal's status bar — you can see what the agent is doing without reading the full terminal output.

Supported events

Ouijit listens for standard Claude Code hook events including tool use notifications and agent lifecycle events. The status display shows a summary of the agent's current action.
