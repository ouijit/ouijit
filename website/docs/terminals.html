Terminal Sessions

Integrated terminal sessions with a card stack UI for multi-terminal workflows.

Card stack

Terminals are displayed as stacked cards on the right side of project mode. The active terminal is in front; background terminals peek out behind it so you can see what's running.

Each terminal is a full PTY session running your default shell, opened in the task's worktree directory.

Opening terminals

When you start a task (move it to In Progress), a terminal opens automatically in the task's worktree. You can open additional terminals for the same task — useful for running a dev server in one terminal and editing in another.

Switching terminals

Click on a background terminal card to bring it to the front, or use keyboard shortcuts:

ShortcutAction
Cmd+] / Ctrl+]Next terminal
Cmd+[ / Ctrl+[Previous terminal
Cmd+T / Ctrl+TNew terminal
Cmd+W / Ctrl+WClose terminal

Terminal features

  • Full xterm.js emulation — colors, cursor positioning, alternate screen buffer, mouse events
  • Output buffering — terminal output is buffered so you don't lose content when switching between terminals
  • Auto-resize — terminal dimensions adjust when the window is resized
  • Git status — each terminal shows the current branch and changed file count in a status bar

Working with CLI agents

Ouijit's terminals work with any CLI tool. Launch Claude Code, Cursor, Aider, or any other agent directly in the terminal. The worktree isolation means each agent works in its own sandbox without interfering with other tasks.
