Terminal Sessions

Integrated terminal sessions with a card stack UI for multi-terminal workflows.

Card stack

Terminals are displayed as stacked cards. The active terminal is in front; background terminals peek out behind it so you can see what's running.

Each terminal is a full PTY session running your default shell, opened in the task's worktree directory.

Opening terminals

When you start a task (move it to In Progress), a terminal opens automatically in the task's worktree. You can open additional terminals for the same task — useful for running a dev server in one and editing in another, or letting multiple AI agents fight to the death.

Switching terminals

Click on a background terminal card to bring it to the front, or use keyboard shortcuts:

ShortcutAction
⌘1–⌘9 / Ctrl+1–Ctrl+9Select terminal by position
⌘⇧← / Ctrl+Shift+LeftPrevious stack page
⌘⇧→ / Ctrl+Shift+RightNext stack page
⌘I / Ctrl+INew terminal
⌘W / Ctrl+WClose terminal

I'd like to let you rebind these, but I haven't gotten around to it yet.

Working with CLI agents

Ouijit's terminals work with any CLI tool. Launch Claude Code, Cursor, Aider, or any other agent directly in the terminal. The worktree isolation means each agent works in its own sandbox without interfering with other tasks. Only Claude Code gets fancy status lights right now — I play favorites.
