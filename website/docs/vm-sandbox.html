VM Sandbox

Run terminal sessions inside a lightweight Linux VM so agents can't touch what they shouldn't.

Why sandbox?

Git worktrees isolate your code, but a terminal session still has full access to your host — your home directory, credentials, system binaries, network. When CLI agents run arbitrary commands, that's a lot of trust.

The VM sandbox wraps each terminal in a Lima virtual machine. Your project worktree is mounted into the VM (read-write), but the rest of your filesystem is walled off. The agent can install packages, run builds, and execute code without risk to your host.

How it works

  1. Enable sandbox — Click the cube icon in the terminal toolbar to open the sandbox dropdown. Toggle it on.
  2. First terminal — Ouijit creates an Ubuntu 24.04 VM via Lima, starts it, and waits for SSH. This takes a minute or two on first boot; subsequent starts are faster.
  3. Shell sessions — Terminals run inside the VM via limactl shell. Your worktree is mounted at the same path as on the host, so relative paths and git operations work normally.
  4. VM lifecycle — The sandbox dropdown gives you direct control: Start, Stop, Recreate, or Delete the VM. When you open a sandboxed terminal, Ouijit starts the VM if it's stopped, or creates one if it doesn't exist yet.

Setup hook

VMs start from a clean Ubuntu image with bash, git, curl, nodejs, npm, python3, and build-essential pre-installed. If your project needs more, use the sandbox-setup hook.

The setup hook runs once per VM session (not per terminal). If you edit the hook script, it re-runs automatically on the next terminal open.

Configure it from the sandbox dropdown — click the gear icon next to "Setup Hook", or create a new one if none exists.

# Example: install project dependencies inside the VM
cd /path/to/your/worktree
npm install

Resource configuration

Each project's VM can be configured with custom resource limits from the sandbox dropdown:

  • Memory — default 4 GiB
  • Disk — default 10 GiB
  • CPUs — default 2

Changes to resources require recreating the VM (the dropdown has a "Recreate VM" option for this).

Claude Code integration

Ouijit automatically injects its hook helper script and Claude Code settings into each sandboxed shell. This means Claude Code's hook status events (thinking, tool use, idle) work inside the VM just like they do on the host — the VM shell communicates back to Ouijit via host.lima.internal.

Requirements

  • macOS — Uses Apple's Virtualization.framework (vz) with vzNAT networking. No extra installation needed if Lima is available.
  • Linux — Uses QEMU with user-mode networking.

Ouijit bundles limactl, so there's nothing extra to install.

Limitations

  • First boot downloads an Ubuntu cloud image and provisions packages — requires internet and takes 1-2 minutes
  • GPU passthrough is not supported
  • macOS-specific tooling (Xcode, Swift) is not available inside the Linux VM
  • Docker-in-VM requires additional Lima configuration
  • The VM home directory is ephemeral — anything outside mounted directories is lost on recreate
