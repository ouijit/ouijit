VM Sandbox

Run terminal sessions in a Lima virtual machine for full isolation from your host.

Overview

When working with CLI agents that execute arbitrary commands, you may want stronger isolation than git worktrees alone provide. Ouijit integrates with Lima to run terminal sessions inside a lightweight Linux VM.

The VM shares your project directory (read/write mount) but isolates the rest of your file system, network, and system processes.

Setup

Install Lima

# macOS (Homebrew)
brew install lima

# Linux
# See https://lima-vm.io/docs/installation/

Enable in Ouijit

Open your project settings and toggle VM Sandbox on. Ouijit will create and manage a Lima VM instance for the project.

How it works

  1. When VM sandbox is enabled, Ouijit generates a Lima YAML configuration for the project.
  2. On first terminal open, the VM is started (this takes a few seconds on first boot).
  3. Terminal sessions run inside the VM via limactl shell.
  4. Your project worktree is mounted into the VM at the same path, so tools and relative paths work as expected.

When to use it

  • Untrusted agents — if an agent might run destructive commands, the VM limits blast radius
  • Dependency isolation — install project dependencies inside the VM without affecting your host
  • Reproducible environments — consistent Linux environment regardless of host OS

Limitations

  • VM startup adds a few seconds to the first terminal open
  • GPU passthrough is not supported
  • Docker-in-VM requires additional Lima configuration
  • macOS-specific tooling (Xcode, Swift) is not available inside the Linux VM
