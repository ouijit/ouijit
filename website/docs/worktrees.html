Worktree Isolation

Every task gets its own git worktree, so you can work on multiple features simultaneously without branch switching.

What are git worktrees?

Git worktrees let you check out multiple branches of the same repository into separate directories at the same time. Instead of stashing changes and switching branches, each worktree has its own working directory with its own checked-out branch.

Ouijit manages this automatically: when you start a task, it creates a worktree; when the task is done, it cleans up.

How Ouijit uses worktrees

Task lifecycle

  1. Create task — A new branch is created from your main branch (e.g. T-42).
  2. Start task — Ouijit creates a git worktree in a worktrees/ directory alongside your main repo. The terminal opens in this isolated directory.
  3. Work — All changes happen in the worktree. Your main branch stays clean.
  4. Review — Use the diff panel to see what changed relative to main.
  5. Complete — Merge the branch back and remove the worktree.

Copy-on-Write cloning

Creating a full copy of a large repository for each task would be slow and waste disk space. Ouijit uses Copy-on-Write (CoW) cloning via the native file system (APFS on macOS, reflinks on Linux with btrfs/XFS).

CoW clones are nearly instant and share disk blocks with the original until a file is modified. This means:

  • Worktree creation is fast regardless of repo size
  • Disk usage is proportional to your actual changes, not the full repo
  • The node_modules/ directory is cloned too, so dependencies are ready immediately

Directory structure

my-project/              # Main repo (main branch)
worktrees/
  my-project/
    T-42/                # Worktree for task 42
    T-43/                # Worktree for task 43

Worktree directories are siblings of your main repo in a shared parent worktrees/ folder.

Manual worktree operations

Ouijit manages worktrees for you, but they're standard git worktrees. You can inspect them with regular git commands:

# List all worktrees
git worktree list

# From inside a worktree, check which branch it's on
git branch --show-current
