Kanban Board

A four-column board for organizing tasks with drag-and-drop.

Columns

The board has four fixed columns that map to a task's lifecycle:

ColumnMeaning
To DoPlanned work. No worktree or branch created yet.
In ProgressActive work. A worktree and branch are created when a task moves here.
In ReviewWork is done, ready for review. Worktree is still active.
DoneCompleted. The branch can be merged and the worktree removed.

Moving a task between columns triggers lifecycle hooks — scripts that can do anything: run tests, open a PR, or kick off an agent.

Creating tasks

Click the + button on any column header, or press ⌘N (Ctrl+N on Linux) to create a new task. Give it a title and optional description.

Each task gets an auto-incrementing number (e.g. T-1, T-2) used for branch names and worktree directories. I am still questioning this decision.

Drag and drop

Drag task cards between columns to change their status. You can also reorder tasks within a column.

When dragging a task from To Do to In Progress, Ouijit automatically creates a git worktree and branch for the task.

Task actions

Right-click a task card to access its context menu:

  • Open in Terminal — Launch a terminal in the task's worktree
  • Open in Sandbox — Launch a sandboxed terminal (if Lima is available)
  • Open in Editor — Open the worktree in your editor
  • Close / Reopen — Archive or reopen a task

If a task already has running terminals, they'll appear at the top of the menu so you can jump straight to them.
